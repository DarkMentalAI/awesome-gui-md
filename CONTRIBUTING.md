# Contributing to Awesome GUI.md

Thanks for helping improve Awesome GUI.md.

This project collects interface contracts and GUI case patterns. Contributions must explain structure, state, interaction, accessibility, and reuse guidance.

## What We Accept

- New `GUI.md` pattern entries.
- Improvements to existing `GUI.md` or `HTML.md` files.
- Boundary, authoring, or evaluation documentation improvements.
- Corrections that make adapter responsibilities clearer.

## What We Do Not Accept

- Unsourced screenshot dumps.
- Pure visual inspiration with no interface analysis.
- Private, confidential, paywalled, or user-identifying screenshots.
- AI-generated mockups presented as real product references.
- Promotional entries with no reusable interface lesson.
- Framework-specific APIs inside `GUI.md`.

## Entry Requirements

Each pattern entry must include:

- `GUI.md`
- `HTML.md`
- `README.md`
- `meta.json`

Each entry should explain:

- the interface problem
- surface structure
- component contracts
- states
- interactions
- accessibility requirements
- reuse guidance
- caveats

## Change Workflow

All changes must be opened as pull requests and reviewed before merging. Do not push directly to `main`.

## Pull Request Checklist

- [ ] The contribution keeps `GUI.md` implementation-neutral.
- [ ] Web-specific rules live in `HTML.md`.
- [ ] The entry explains why the pattern matters.
- [ ] The entry documents states and interactions.
- [ ] The entry includes accessibility requirements.
- [ ] The entry includes reuse guidance and caveats.
- [ ] The contribution avoids private or user-identifying material.
- [ ] The contribution avoids marketing language.

## Screenshot and Source Policy

Screenshots are acceptable only for commentary, critique, documentation, or educational reference. Do not include private data. Blur or remove personal information. Prefer self-created abstract pattern examples for the first version.

## Maintenance

Entries may be marked stale when the pattern no longer represents current guidance. Duplicate entries should be merged unless they teach different interaction lessons.
